# 🕹️ 오락실 테트리스 웹앱 — 설계도 (Claude Code 작업용)

> 이 문서를 프로젝트 루트에 `DESIGN.md` 또는 `CLAUDE.md`로 넣고 Claude Code에게
> "이 설계도대로 단계별로 만들어줘"라고 지시하면 됩니다.
> 각 단계는 **테스트 가능한 단위**로 쪼개져 있고, 되돌리기 쉬운 순서로 배치돼 있습니다.

| 항목 | 내용 |
|------|------|

## 1. 기술 스택 (권장)

| 영역 | 선택 | 이유 |
|------|------|------|
| 언어 | **TypeScript** | 게임 상태가 복잡해서 타입이 버그를 크게 줄여줌 |
| 빌드 | **Vite** | 가볍고 빠름. PWA 플러그인 지원 |
| 렌더링 | **HTML5 Canvas (바닐라, 게임 프레임워크 없음)** | 60fps 게임 루프에 React 리렌더링은 부적합. Canvas가 모바일 성능 최고 |
| 오디오 | **Web Audio API (칩튠 직접 합성)** | 음원 파일 0개, 저작권 깨끗, 용량 최소 |
| UI(메뉴) | **순수 HTML/CSS** (게임 보드만 Canvas) | 메뉴·점수판은 DOM이 편함 |
| 배포 | Vercel / Netlify / GitHub Pages | 정적 호스팅이면 어디든 OK |

> **React를 쓰고 싶다면**: 메뉴·설정 화면만 React로, 게임 보드는 `<canvas>` + 별도 게임 엔진 클래스로 분리하세요. 게임 루프는 절대 React state로 돌리지 말 것(프레임 드랍·리렌더 폭발).

## 2. 테트리스 게임 메커니즘 분석 (표준 규칙)

### 2.1 보드
- 가로 **10칸** × 세로 **20칸**(보이는 영역) + 위쪽 버퍼 2~4줄(피스 생성 공간)
- 좌표계: 좌상단 (0,0), 오른쪽 +x, 아래쪽 +y

### 2.2 7가지 테트로미노
```
I  ▢▢▢▢      O  ▢▢       T  ▢▢▢      S   ▢▢     Z  ▢▢
                ▢▢          ▢          ▢▢          ▢▢

J  ▢          L    ▢
   ▢▢▢            ▢▢▢
```
- 각 피스는 4×4(또는 I·O 전용) 회전 행렬로 표현
- 표준 색상: I=하늘, O=노랑, T=보라, S=초록, Z=빨강, J=파랑, L=주황

### 2.3 핵심 규칙
| 규칙 | 설명 |
|------|------|
| **7-bag 랜덤** | 7개 피스를 한 묶음으로 섞어 순서대로 배출 → 같은 피스가 연속으로 몰리지 않음 |
| **SRS 회전** | Super Rotation System. 회전 시 벽/블록에 막히면 **월킥**(wall kick)으로 살짝 밀어 회전 시도 |
| **고스트 피스** | 현재 피스가 떨어질 위치를 반투명으로 미리 표시 |
| **하드 드롭** | 즉시 바닥까지 떨어뜨림 |
| **소프트 드롭** | 빠르게 내림(가속) |
| **홀드(Hold)** | 피스 1개를 보관했다가 나중에 교체. 한 피스당 1회 |
| **락 딜레이** | 바닥에 닿아도 약 0.5초간 이동/회전 가능(고수 플레이 핵심) |
| **줄 삭제** | 가로 한 줄이 꽉 차면 삭제, 위 블록이 내려옴 |

### 2.4 점수 시스템 (표준 가이드라인 기반)
| 액션 | 점수 (레벨 L 기준) |
|------|------|
| 싱글(1줄) | 100 × L |
| 더블(2줄) | 300 × L |
| 트리플(3줄) | 500 × L |
| **테트리스(4줄)** | 800 × L |
| 소프트 드롭 | 1점 / 칸 |
| 하드 드롭 | 2점 / 칸 |
| T-스핀 싱글/더블/트리플 | 800 / 1200 / 1600 × L |
| **B2B(연속 테트리스/T-스핀)** | ×1.5 보너스 |
| **콤보** | 연속 줄삭제마다 50 × 콤보수 × L |

## 3. ⭐ 100단계 난이도 설계 (이 프로젝트의 핵심)

> **문제**: 표준 테트리스는 약 레벨 15~20에서 이미 "즉시 낙하(20G)"에 도달합니다.
> 속도만으로는 100단계를 못 만듭니다. → **여러 축의 난이도를 단계별로 조합**합니다.

### 3.1 난이도 7대 축
| 축 | 1단계 | 100단계 |
|----|-------|---------|
| ① 중력(낙하 속도) | 1칸/1초 | 즉시 낙하(20G) |
| ② 락 딜레이 | 500ms | 16ms(1프레임) |
| ③ 넥스트 미리보기 | 5개 | 0개 |
| ④ 고스트 피스 | 항상 ON | OFF |
| ⑤ 가비지 라인(밑에서 차오름) | 없음 | 빠르게 차오름 |
| ⑥ 홀드 기능 | 자유 | 제한/봉인 |
| ⑦ 특수 기믹 | 없음 | 블록 페이드·보드 흔들림 등 |

### 3.2 5개 페이즈 구조 (각 20단계)
| 페이즈 | 단계 | 테마 | 주요 변화 |
|--------|------|------|-----------|
| **P1 입문** | 1–20 | 정통 테트리스 | 중력만 점진 증가. 모든 보조(고스트·홀드·넥스트5) 풀가동 |
| **P2 가속** | 21–40 | 손이 바빠짐 | 중력 거의 1G→1G, 락 딜레이 500→300ms, 넥스트 5→3 |
| **P3 압박** | 41–60 | 즉시 낙하 진입 | 20G 도달, 가비지 라인 등장 시작, 고스트 깜빡임, 락 딜레이 300→150ms |
| **P4 혼돈** | 61–80 | 보조 박탈 | 가비지 가속, 넥스트 3→1, 고스트 OFF, 홀드 간헐 봉인 |
| **P5 광기** | 81–100 | 보스전 | 놓인 블록 서서히 사라짐(페이드), 넥스트 0, 락 딜레이 16ms. 100단계는 끝판왕 |

### 3.3 단계별 수치 (대표 체크포인트)
> 전체 100줄 표 대신 대표 단계 + **공식**을 제공합니다. Claude Code가 공식으로 100단계를 자동 생성하게 하세요.

| 단계 | 중력(초/칸) | 락딜레이(ms) | 넥스트 | 고스트 | 가비지 | 비고 |
|------|-----------|------------|--------|--------|--------|------|
| 1 | 1.000 | 500 | 5 | ON | — | 시작 |
| 5 | 0.360 | 500 | 5 | ON | — | |
| 10 | 0.092 | 500 | 5 | ON | — | |
| 15 | 0.021 | 500 | 4 | ON | — | |
| 20 | 0.0036 | 500 | 3 | ON | — | 거의 즉시 |
| 30 | 즉시(20G) | 400 | 3 | ON | — | |
| 40 | 즉시 | 300 | 3 | ON | — | |
| 50 | 즉시 | 220 | 3 | 깜빡임 | 30초마다 1줄 | |
| 60 | 즉시 | 150 | 2 | 깜빡임 | 20초마다 1줄 | |
| 70 | 즉시 | 110 | 2 | OFF | 15초마다 1줄 | |
| 80 | 즉시 | 60 | 1 | OFF | 12초마다 1줄 | 홀드 간헐봉인 |
| 90 | 즉시 | 40 | 1 | OFF | 10초마다 1줄 | 블록 페이드 |
| 100 | 즉시 | 16 | 0 | OFF | 8초마다 1줄 | 끝판왕 |

### 3.4 난이도 생성 공식 (Claude Code가 이걸로 100단계 자동 생성)
```typescript
// config/difficulty.ts
export interface LevelConfig {
  level: number;
  gravity: number;       // 초/칸 (0이면 즉시 낙하 20G)
  lockDelayMs: number;
  nextCount: number;     // 미리보기 개수
  ghost: 'on' | 'flicker' | 'off';
  garbageIntervalSec: number | null; // null이면 가비지 없음
  holdMode: 'free' | 'limited' | 'locked';
  fadeBlocks: boolean;   // 놓인 블록 서서히 사라짐
}

export function getLevelConfig(level: number): LevelConfig {
  // ① 중력: 표준 가이드라인 공식 (1~19), 20부터 즉시 낙하
  const gravity = level >= 20
    ? 0
    : Math.pow(0.8 - (level - 1) * 0.007, level - 1);

  // ② 락 딜레이: 20단계까지 500ms 고정, 이후 선형 감소 → 100단계 16ms
  const lockDelayMs = level <= 20
    ? 500
    : Math.max(16, Math.round(500 - (level - 20) * (484 / 80)));

  // ③ 넥스트 미리보기
  const nextCount =
    level <= 14 ? 5 :
    level <= 40 ? 3 :
    level <= 65 ? 2 :
    level <= 95 ? 1 : 0;

  // ④ 고스트
  const ghost: LevelConfig['ghost'] =
    level <= 45 ? 'on' :
    level <= 65 ? 'flicker' : 'off';

  // ⑤ 가비지 라인
  const garbageIntervalSec =
    level < 50 ? null :
    Math.max(8, 30 - Math.floor((level - 50) / 5) * 2);

  // ⑥ 홀드
  const holdMode: LevelConfig['holdMode'] =
    level < 76 ? 'free' :
    level < 96 ? 'limited' : 'locked';

  // ⑦ 블록 페이드
  const fadeBlocks = level >= 86;

  return { level, gravity, lockDelayMs, nextCount, ghost,
           garbageIntervalSec, holdMode, fadeBlocks };
}
```

### 3.5 단계 진행 방식 (2가지 모드 동시 지원)
1. **레벨 선택 모드**: 시작 화면에서 1~100 중 골라 시작 (도전 모드)
2. **무한 진행 모드**: 줄 일정 개수 삭제 시 자동 레벨업 (P1=10줄/레벨, P3=8줄, P5=6줄 등)

## 4. 📱 모바일 조작 설계 (요구사항 핵심)

> "한 손으로도 쉽게" + "오작동 적게"가 목표. **2가지 입력 방식을 동시 제공**하고 설정에서 전환.

### 4.1 방식 A — 제스처(기본)
| 동작 | 제스처 |
|------|--------|
| 좌/우 이동 | 화면 좌우 **스와이프** (드래그 거리에 비례해 여러 칸 이동) |
| 회전 | **탭(짧게 터치)** |
| 소프트 드롭 | 아래로 **천천히 스와이프** |
| 하드 드롭 | 아래로 **빠르게 플릭(휙)** |
| 홀드 | **위로 스와이프** |

### 4.2 방식 B — 가상 버튼(D-pad)
```
          [회전]
[◀]  [▼ 소프트]  [▶]
       [⏬ 하드드롭]   [HOLD]
```
- 화면 하단에 반투명 버튼. 누르고 있으면 연속 이동(DAS/ARR 적용)
- 버튼 크기 최소 48×48px(터치 가이드라인), 엄지 닿는 범위에 배치

### 4.3 모바일 UX 디테일 (꼭 넣을 것)
| 항목 | 처리 |
|------|------|
| 햅틱 피드백 | 회전·착지·줄삭제 시 `navigator.vibrate()` (짧게) |
| 화면 회전 | 세로 모드 기본. 가로 모드 시 좌우 컨트롤 분리 배치 |
| 더블탭 줌 방지 | `touch-action: none`, `user-select: none` |
| 풀스크린/PWA | 홈 화면 추가 시 주소창 없이 풀스크린 |
| 화면 꺼짐 방지 | Wake Lock API로 플레이 중 화면 유지 |
| DAS/ARR 설정 | 좌우 길게 누를 때 첫 반복 지연(DAS)·반복 간격(ARR) 조절 가능 |
| 반응형 | 보드 크기를 화면 높이에 맞춰 자동 스케일 |

## 5. 🎵 오디오 / BGM 모듈 (칩튠 직접 합성)

### 5.1 방침
- **음원 파일 금지**. Web Audio API 오실레이터로 실시간 합성 → 저작권 깨끗 + 용량 0
- 멜로디 = **코로베이니키(퍼블릭 도메인)**. 아래 연주 데이터를 직접 넣음
- 음색: 사각파(square) = 멜로디, 삼각파(triangle) = 베이스 → NES/오락실 느낌

### 5.3 효과음(SFX)도 합성으로
| 이벤트 | 소리 |
|--------|------|
| 이동 | 아주 짧은 클릭(노이즈/사각파 10ms) |
| 회전 | 약간 높은 핑 |
| 착지(lock) | 낮은 둔탁음 |
| 줄삭제 | 상승 아르페지오 |
| 테트리스(4줄) | 화려한 상승음 + 햅틱 강하게 |
| 레벨업 | 팡파레 짧게 |
| 게임오버 | 하강 글리산도 |

### 5.4 주의: 모바일 오디오 정책
- iOS/안드로이드는 **사용자 첫 터치 후**에만 오디오 재생 가능
  → "탭하여 시작" 화면에서 `AudioContext.resume()` 호출
- BGM ON/OFF 토글 + 볼륨 슬라이더 제공

```typescript
// 보드: 0=빈칸, 1~7=피스 색상 인덱스, 8=가비지
type Cell = 0|1|2|3|4|5|6|7|8;
type Board = Cell[][];            // [y][x], 20행 × 10열

interface ActivePiece {
  type: 'I'|'O'|'T'|'S'|'Z'|'J'|'L';
  rotation: 0|1|2|3;
  x: number;  y: number;          // 좌상단 기준 위치
}

interface GameState {
  board: Board;
  active: ActivePiece;
  hold: ActivePiece['type'] | null;
  holdUsed: boolean;
  nextQueue: ActivePiece['type'][];
  score: number;
  lines: number;
  level: number;                  // 1~100
  combo: number;
  backToBack: boolean;
  status: 'ready'|'playing'|'paused'|'gameover';
  config: LevelConfig;            // 현재 레벨 설정
}
```

## 8. 게임 루프 아키텍처

```
requestAnimationFrame 루프 (목표 60fps)
  │
  ├─ 1. 입력 처리 (이번 프레임에 들어온 터치/키)
  ├─ 2. 중력 적용 (config.gravity 만큼 누적 → 1칸 낙하 판정)
  ├─ 3. 락 딜레이 처리 (바닥 닿음 + 타이머 → 고정)
  ├─ 4. 줄삭제 검사 → 점수 → 레벨업 검사 → config 갱신
  ├─ 5. 가비지 라인 타이머 (해당 레벨이면)
  ├─ 6. 특수 기믹 (페이드 등)
  └─ 7. 렌더링 (Canvas 다시 그리기 + HUD 갱신)
```
> **고정 timestep 권장**: 기기마다 fps가 달라도 게임 속도가 같도록 누적시간(accumulator) 방식 사용.

## 9. ⭐ Claude Code 단계별 작업 프롬프트 (복사해서 사용)

> 각 단계가 끝나면 **반드시 브라우저에서 동작 확인** 후 다음으로. 되돌리기 쉬운 순서입니다.

### 단계 0 — 프로젝트 세팅
```
Vite + TypeScript로 빈 프로젝트를 만들어줘.
DESIGN.md(이 설계도)를 읽고 6번 "파일/모듈 구조"대로 빈 파일/폴더 골격만 먼저 만들어줘.
아직 로직은 비우고, index.html에 <canvas>와 기본 레이아웃만. npm run dev로 빈 화면이 뜨는 것까지 확인.
```

### 단계 1 — 보드 + 피스 렌더링 (입력·중력 없음)
```
src/game/Board.ts와 Piece.ts를 구현해줘.
7가지 테트로미노 정의, 보드 10×20 그리드, CanvasRenderer로 보드와 고정된 피스 하나를 그려줘.
이 단계 목표: 화면에 격자와 색깔 블록이 보이는 것. 움직임은 아직 없음.
```

### 단계 2 — 키보드 조작 + 중력 + 줄삭제 (PC에서 먼저 완성)
```
GameEngine.ts에 게임 루프를 만들어줘. 8번 "게임 루프 아키텍처"와 7번 데이터 구조를 따라줘.
- 좌우 이동, 회전(일단 단순 회전), 소프트/하드 드롭, 중력(고정 1초/칸)
- 바닥/블록 충돌 판정, 줄삭제, 줄삭제 시 위 블록 내려오기
- 7-bag 랜덤(bag.ts), 넥스트 큐 5개, 홀드 기능
키보드(← → ↑ ↓ Space Shift)로 PC에서 플레이되게. 점수는 아직 단순하게.
```

### 단계 3 — SRS 회전 + 고스트 + 정통 점수
```
srs.ts에 SRS 월킥 테이블을 구현해서 회전을 정식 규칙으로 바꿔줘.
고스트 피스(render/ghost.ts), 그리고 scoring.ts에 2.4 표준 점수(테트리스·콤보·B2B·T스핀)를 적용해줘.
```

### 단계 4 — ★ 100단계 난이도 시스템
```
config/difficulty.ts에 설계도 3.4의 getLevelConfig(level) 공식을 그대로 구현해줘.
GameEngine이 매 레벨업마다 이 config를 읽어서 중력·락딜레이·넥스트개수·고스트·가비지·홀드·페이드를 반영하게 해줘.
시작 화면(StartScreen.ts)에서 "레벨 선택 모드(1~100)"와 "무한 진행 모드"를 고를 수 있게.
가비지 라인(garbage.ts)과 블록 페이드 기믹도 이 단계에서 구현.
테스트: 50단계, 80단계, 100단계로 시작해서 각 기믹이 실제로 켜지는지 확인.
```

### 단계 5 — ★ 모바일 터치 조작
```
설계도 4번대로 모바일 조작을 구현해줘.
- TouchInput.ts: 제스처 방식(스와이프 이동/탭 회전/플릭 하드드롭/위로 홀드)
- ButtonInput.ts: 화면 하단 반투명 가상 버튼(D-pad)
- 설정에서 두 방식 전환
- 햅틱(vibrate), touch-action:none, 화면 높이에 맞춘 반응형 스케일, Wake Lock
실제 휴대폰(또는 크롬 모바일 에뮬레이션)에서 한 손 조작이 자연스러운지 확인.
```

### 단계 6 — ★ 칩튠 BGM + 효과음
```
설계도 5번대로 audio 모듈을 구현해줘.
- ChiptuneEngine.ts: Web Audio 오실레이터(사각파 멜로디 + 삼각파 베이스)
- melody.ts의 코로베이니키 데이터를 루프 재생, 레벨 오를수록 템포 빨라지게
- sfx.ts: 이동/회전/착지/줄삭제/테트리스/레벨업/게임오버 효과음
- "탭하여 시작" 화면에서 AudioContext.resume() 호출(모바일 정책)
- BGM/SFX ON-OFF 토글, 볼륨 슬라이더
```

### 단계 7 — 마무리(PWA·점수저장·연출)
```
- PWA 설정(manifest.json, 서비스워커)으로 홈 화면 추가 가능하게
- 최고 점수 localStorage 저장
- 줄삭제 애니메이션, 레벨업 연출, 게임오버 화면
- 오락실 감성 CRT 스캔라인 효과(CSS, 선택사항)
```
